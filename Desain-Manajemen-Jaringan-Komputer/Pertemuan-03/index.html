Pekan 3: VLAN, Trunking, dan Router-on-a-Stick

Capaian Pembelajaran

  • Memahami konsep dasar VLAN dan trunking pada switch.
  • Dapat mengonfigurasi VLAN (access port) dan trunk port pada switch.
  • Mampu menerapkan router-on-a-stick (sub-interface VLAN) pada router untuk menghubungkan beberapa VLAN.
  • Menguasai uji konektivitas lintas VLAN melalui perintah ping dan troubleshooting dasar.

Konsep Dasar VLAN dan Trunking

VLAN (Virtual LAN) adalah pemisahan logis dari jaringan fisik untuk memisahkan domain siaran dan meningkatkan keamanan serta performa jaringan.

  • VLAN: Mengelompokkan perangkat berdasarkan fungsi atau lokasi tanpa bergantung pada fisik jaringan.
  • Trunking: Teknik untuk mengirimkan trafik dari beberapa VLAN melalui satu link fisik antar switch.

Konfigurasi VLAN dan Trunk Port

Pada switch, konfigurasi VLAN dilakukan pada port access untuk menghubungkan perangkat ke VLAN tertentu.

  • Access Port: Port yang menghubungkan perangkat end-user ke satu VLAN.
  • Trunk Port: Port yang membawa trafik dari beberapa VLAN, biasanya digunakan untuk interkoneksi antar switch.

Contoh perintah: switchport mode access dan switchport mode trunk

Router-on-a-Stick

Router-on-a-Stick adalah teknik untuk menghubungkan beberapa VLAN melalui satu router dengan membuat sub-interface untuk tiap VLAN.

  • Setiap sub-interface diberi IP address yang berbeda sesuai dengan subnet VLAN.
  • Menggunakan encapsulation dot1q untuk menandai trafik VLAN.

Contoh perintah pada router: interface GigabitEthernet0/0.10 dengan encapsulation dot1Q 10

Uji Konektivitas dan Troubleshooting

Setelah konfigurasi, penting untuk melakukan pengujian konektivitas lintas VLAN.

  • Gunakan perintah ping untuk menguji komunikasi antar perangkat di VLAN yang berbeda.
  • Jika terjadi masalah, periksa konfigurasi VLAN, trunk, dan sub-interface pada router.
  • Gunakan perintah show vlan dan show interfaces trunk untuk verifikasi.
