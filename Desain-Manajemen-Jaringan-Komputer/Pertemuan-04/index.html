Pekan 4: Routing dan Static Routing

Capaian Pembelajaran

  • Mahasiswa dapat menjelaskan konsep dasar routing serta perbedaan antara static route dan dynamic route.
  • Mahasiswa memahami cara membaca dan memverifikasi routing table pada router.
  • Mahasiswa mampu mengkonfigurasi static routing pada perangkat jaringan (router Cisco) sesuai topologi yang telah ditentukan.
  • Mahasiswa dapat menguji konektivitas antar subnet/VLAN dengan menggunakan perintah ping.

Konsep Dasar Routing

Routing adalah proses pengiriman paket data dari sumber ke tujuan melalui jaringan. Routing menentukan jalur terbaik berdasarkan tabel routing.

  • Static Route: Rute yang dikonfigurasi secara manual oleh administrator. Tidak berubah kecuali diubah secara manual.
  • Dynamic Route: Rute yang dipelajari dan diperbarui secara otomatis oleh protokol routing (misalnya OSPF, EIGRP, RIP).

Memverifikasi Routing Table

Routing table adalah basis data yang menyimpan informasi jalur untuk mencapai jaringan tujuan.

  • Gunakan perintah show ip route pada router Cisco untuk melihat tabel routing.
  • Periksa rute yang terdaftar, metrik, dan antarmuka keluar untuk memastikan konfigurasi benar.

Konfigurasi Static Routing

Static routing dikonfigurasi secara manual pada router untuk mengarahkan lalu lintas ke subnet/VLAN tertentu.

  • Contoh perintah: ip route [destination_network] [subnet_mask] [next_hop_address]
  • Pastikan routing table diperbarui dan sesuai dengan topologi jaringan.

Jawaban Studi Kasus: Konfigurasi Static Routing

Berdasarkan topologi, berikut langkah-langkah konfigurasi static routing pada Router A dan Router B:

  • Router A: 
        ! IP interface ke PC A: 192.168.2.1
    ! IP interface ke Router B: 10.2.0.2

    ip route 192.168.1.0 255.255.255.0 10.2.0.1
    Menambahkan rute statis agar Router A mengetahui jaringan 192.168.1.0/24 melalui Router B (10.2.0.1).
  • Router B: 
        ! IP interface ke PC B: 192.168.1.1
    ! IP interface ke Router A: 10.2.0.1

    ip route 192.168.2.0 255.255.255.0 10.2.0.2
    Menambahkan rute statis agar Router B mengetahui jaringan 192.168.2.0/24 melalui Router A (10.2.0.2).

Setelah konfigurasi, lakukan uji konektivitas dengan perintah ping dari PC A (192.168.2.10) ke PC B (192.168.1.10) dan sebaliknya.

Jika ping berhasil, berarti routing statis telah berfungsi dengan baik.
