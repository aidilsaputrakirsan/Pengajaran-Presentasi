Pekan 6: DHCP, DNS, NAT/PAT, dan Firewall

Capaian Pembelajaran

  • Memahami cara kerja DHCP, DNS, serta peran NAT/PAT dalam penggunaan IP publik dan privat.
  • Mampu mengonfigurasi layanan DHCP dan DNS di simulator jaringan agar perangkat klien menerima IP dan melakukan resolusi nama.
  • Mampu mengimplementasikan NAT (static/PAT) pada router agar host di jaringan lokal terhubung ke jaringan eksternal.
  • Memahami firewall dasar di router (ACL/zone-based) dan membuat rule sederhana untuk membatasi akses.

Konsep Dasar Layanan Jaringan

  • DHCP: Protokol yang mengalokasikan IP secara otomatis kepada perangkat klien.
  • DNS: Sistem yang mengonversi nama domain menjadi alamat IP agar mudah diingat.
  • NAT/PAT: Teknik untuk menerjemahkan alamat IP privat ke IP publik, memungkinkan akses ke internet.
  • Firewall (ACL/Zone-Based): Pengaturan aturan akses untuk membatasi trafik jaringan.

Konfigurasi DHCP dan DNS

Contoh konfigurasi di simulator (Cisco Packet Tracer/GNS3):

  • DHCP: Mengatur pool alamat IP, subnet mask, default gateway, dan DNS server.
  • DNS: Menetapkan nama domain dan menghubungkannya dengan alamat IP tertentu.

Perangkat klien yang terhubung ke jaringan akan mendapatkan IP secara otomatis dan dapat menyelesaikan nama domain.

Implementasi NAT/PAT

NAT/PAT memungkinkan host di jaringan privat mengakses internet dengan menerjemahkan IP lokal ke IP publik.

  • NAT Static: Satu-ke-satu mapping antara IP privat dan IP publik.
  • NAT PAT (Overloading): Banyak IP privat menggunakan satu IP publik dengan port berbeda.

Contoh perintah (pada router Cisco): ip nat inside source list 1 interface GigabitEthernet0/1 overload

Firewall Dasar dengan ACL/Zone-Based

  • ACL: Mengizinkan atau memblokir trafik berdasarkan alamat IP, protokol, dan port.
  • Zone-Based Firewall: Mengelompokkan antarmuka ke dalam zona dan menerapkan aturan untuk mengontrol trafik antar zona.

Contoh perintah ACL: access-list 101 permit ip any any (disesuaikan dengan kebutuhan keamanan)
