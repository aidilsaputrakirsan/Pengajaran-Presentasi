Pekan 4: Menghubungkan React ke Flask

🔄 Mengapa Perlu Integrasi Frontend & Backend?

  • Frontend (React) membutuhkan data real-time dari Backend (Flask).
  • Memungkinkan aplikasi untuk dinamis dan interaktif.

Ilustrasi: Alur Integrasi Frontend dan Backend

📡 Tools untuk Integrasi

  • Fetch API (bawaan browser) atau Axios (library) untuk HTTP request di React.
  • Endpoint API di Flask harus siap untuk menerima dan mengirim data.

Tools untuk Integrasi Frontend dan Backend

🔗 Alur Permintaan Data

  1. Client (React) mengirim permintaan ke Server (Flask).
  2. Server (Flask) mengakses Database (PostgreSQL) dan mengembalikan data ke Client.

Diagram Alur Permintaan Data

📝 Modul Praktikum

🔄 Deskripsi Singkat

React (Vite) akan melakukan request ke endpoint Flask, dan data produk dari PostgreSQL akan ditampilkan di frontend. Mahasiswa akan belajar cara melakukan HTTP request (GET) dari React ke Flask dan menampilkan data tersebut di UI React.

🎯 Tujuan Pembelajaran

  • Mampu melakukan HTTP request (GET) dari React ke Flask.
  • Menampilkan data produk di UI React.

👩‍💻 Pembagian Tugas Tim

  • Frontend Engineer: Kode fetch/axios untuk data produk.
  • Backend Engineer: Pastikan endpoint /api/data GET berjalan.

Instalasi Axios (Opsional)


    npm install axios

Modifikasi App.jsx


// src/App.jsx
import React, { useState, useEffect } from 'react';
import axios from 'axios';

function App() {
  const [apiData, setApiData] = useState(null);

  useEffect(() => {
    axios.get('http://localhost:5000/api/data')
      .then(response => setApiData(response.data.data))
      .catch(error => console.error(error));
  }, []);



export default App;

Menjalankan Aplikasi Flask


    cd backend
    source venv/bin/activate  # Linux/MacOS
    # .\venv\Scripts\activate  # Windows
    python app.py

Pastikan Flask berjalan di http://localhost:5000.

Menjalankan Aplikasi React


    cd ../frontend/my-react-app
    npm run dev

Akses React di http://localhost:5173/.

📊 Kriteria Penilaian

  • React menampilkan data dari endpoint Flask (50%).
  • Tidak muncul error CORS (25%).
  • Error handling minimal (25%).

✅ Checklist Praktikum

  • Endpoint /api/data GET berjalan.
  • React menampilkan data dari DB.
  • Minimal error handling (console.error).

🙋‍♂️ Latihan Mandiri

  1. Tambahkan form (POST) dari React ke Flask.
  2. Styling tampilan dengan CSS.
