Pekan 6: Dockerization Bagian 1 (Membuat Dockerfile untuk Flask)

🐳 Mengapa Docker?

  • Docker mempermudah pengelolaan lingkungan pengembangan dan produksi.
  • Isolasi aplikasi dan dependensi dalam container.

Ilustrasi: Docker dan Container

📦 Apa Itu Dockerfile?

  • Dockerfile adalah script yang berisi instruksi untuk membangun Docker image.
  • Mengautomasi proses build dan konfigurasi aplikasi.

Contoh Struktur Dockerfile

🚀 Manfaat Dockerization

  • Konsistensi lingkungan antara development dan production.
  • Mudah dalam scaling dan deployment.
  • Mengurangi konflik dependensi antar proyek.

📝 Modul Praktikum

🔄 Deskripsi Singkat

Mahasiswa akan belajar membuat Dockerfile untuk aplikasi Flask, sehingga aplikasi dapat dijalankan dalam container. Fokus pada pembuatan Dockerfile, build image, dan menjalankan container.

🎯 Tujuan Pembelajaran

  • Memahami sintaks dasar Dockerfile.
  • Mampu membangun Docker image untuk aplikasi Flask.

👩‍💻 Pembagian Tugas Tim

  • Backend Engineer: Mengatur Dockerfile Flask.
  • Infrastructure Engineer: Menjalankan container dan menguji image yang dibuat.

Persiapan Awal

Pastikan Docker Desktop telah berjalan sebelum menjalankan perintah Docker.


    # Cek status Docker
    docker info

Membuat Dockerfile


    # backend/Dockerfile
    FROM python:3.9-slim

    WORKDIR /app

    COPY requirements.txt requirements.txt
    RUN pip install --no-cache-dir -r requirements.txt

    COPY . .

    EXPOSE 5000
    CMD ["python", "app.py"]

Menyiapkan requirements.txt


    flask
    flask-cors
    psycopg2-binary

Membangun Docker Image


    cd backend
    docker build -t flask-backend:1.0 .

Pastikan build berjalan tanpa error.

Menjalankan Docker Container


    docker run -d -p 5000:5000 --name flask-container flask-backend:1.0

Cek container berjalan dengan docker ps.

Verifikasi di Browser

Buka http://localhost:5000 untuk memastikan Flask API berjalan di dalam container.

Flask API Berjalan di Docker

📊 Kriteria Penilaian

  • Dockerfile berfungsi dan tidak error (40%).
  • Image dapat berjalan sebagai container (40%).
  • Dokumentasi Dockerfile singkat (20%).

✅ Checklist Praktikum

  • Dockerfile dibuat di folder backend/.
  • Docker image berhasil dibangun tanpa error.
  • Container berjalan di port 5000.
  • Flask API dapat diakses dari host.

🙋‍♂️ Latihan Mandiri

  • Buat tag versi berbeda, misalnya flask-backend:2.0.
  • Push image ke Docker Hub (opsional, jika memiliki akun).

📢 Deadline & Pengumpulan

  • Kumpulkan screenshot container yang berjalan (docker ps) dan respon dari http://localhost:5000.

🛠️ Troubleshooting Tips

  • Port 5000 sudah digunakan: Gunakan port lain seperti 5001.
  • Container tidak berjalan: Periksa logs dengan docker logs flask-container.
  • Error pada Flask: Pastikan semua dependensi terinstal dengan benar.
