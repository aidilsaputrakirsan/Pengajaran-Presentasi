Pekan 7: Dockerization Bagian 2 (Membuat Dockerfile untuk React dengan Vite)

⚙️ Mengapa Docker untuk React?

  • Mengemas aplikasi frontend ke dalam container mempermudah deployment.
  • Menciptakan lingkungan yang konsisten antara development dan production.

Ilustrasi: Docker dan React

📦 Multi-Stage Build dengan Docker

  • Menggunakan multi-stage build untuk optimasi ukuran image.
  • Build aplikasi di satu stage, serve dengan Nginx di stage berikutnya.

Diagram Multi-Stage Build

🚀 Menjalankan React di Docker dengan Nginx

  • Menggunakan Nginx untuk serve aplikasi React secara efisien.
  • Optimasi performa dan keamanan dengan konfigurasi Nginx.

📝 Modul Praktikum

🔄 Deskripsi Singkat

Melanjutkan proses containerization, mahasiswa akan membuat Dockerfile untuk aplikasi React yang menggunakan Vite. Tujuannya adalah menjalankan frontend di dalam container menggunakan Nginx, sehingga aplikasi frontend dapat di-deploy secara efisien dan konsisten.

🎯 Tujuan Pembelajaran

  • Memahami pembuatan Dockerfile untuk aplikasi React berbasis Vite.
  • Mampu membangun Docker image dan menjalankan container React dengan Vite.

👩‍💻 Pembagian Tugas Tim

  • Frontend Engineer: Mengatur Dockerfile React.
  • Infrastructure Engineer: Menjalankan container React dan menguji.

Persiapan Awal

Pastikan Docker Desktop telah berjalan sebelum menjalankan perintah Docker.


    # Cek status Docker
    docker info

Membuat Dockerfile untuk React dengan Vite


    # frontend/my-react-app/Dockerfile
    FROM node:14-alpine AS build

    WORKDIR /app

    COPY package*.json ./
    RUN npm install

    COPY . .
    RUN npm run build

    FROM nginx:stable-alpine
    COPY --from=build /app/dist /usr/share/nginx/html

    EXPOSE 80
    CMD ["nginx", "-g", "daemon off;"]

Membangun Docker Image


    cd frontend/my-react-app
    docker build -t react-frontend-vite:1.0 .

Pastikan build berjalan tanpa error dan folder dist dihasilkan.

Menjalankan Docker Container


    docker run -d -p 3000:80 --name react-container-vite react-frontend-vite:1.0

Cek container berjalan dengan docker ps.

Verifikasi di Browser

Buka http://localhost:3000 untuk memastikan aplikasi React berjalan di dalam container.

React App Berjalan di Docker dengan Nginx

📊 Kriteria Penilaian

  • Dockerfile React dengan Vite berfungsi tanpa error (40%).
  • Image berhasil dibangun dan container berjalan (40%).
  • Struktur Dockerfile bersih dan terdokumentasi (20%).

✅ Checklist Praktikum

  • Dockerfile dibuat di folder frontend/my-react-app/.
  • Docker image berhasil dibangun tanpa error.
  • Container berjalan di port 3000.
  • React App muncul di browser.

🙋‍♂️ Latihan Mandiri

  • Ubah base image Node.js ke versi lain (misalnya node:16-alpine).
  • Tambahkan environment variable di Dockerfile untuk build arguments.

📢 Deadline & Pengumpulan

  • Kumpulkan screenshot container yang berjalan (docker ps) dan tampilan browser React.

🛠️ Troubleshooting Tips

  • Out of Memory: Pastikan sistem memiliki RAM yang cukup atau tambahkan swap.
  • File Not Found: Pastikan Anda berada di direktori my-react-app sebelum menjalankan docker build.
  • Port Konflik: Jika port 3000 sudah digunakan, ubah port yang dipetakan, misalnya -p 4000:80.
  • Folder dist Tidak Ada: Pastikan perintah npm run build berhasil dan menghasilkan folder dist.
