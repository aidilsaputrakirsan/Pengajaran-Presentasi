Pertemuan 2: Dasar Express.js & Struktur Routing

🔌 Mengapa Express.js?

  • Express.js adalah kerangka kerja web minimalis untuk Node.js.
  • Memudahkan pembuatan server & routing.

Ilustrasi: Logo Express.js

🛣️ Konsep Routing

Menentukan bagaimana aplikasi merespons permintaan ke endpoint tertentu.

  • GET untuk mengambil data.
  • POST untuk mengirim data baru.

Alur request/response di Express.

🌐 Studi Kasus E-Commerce

  • Server Express untuk menampilkan produk, menambah data, dsb.
  • Route GET: ambil produk, Route POST: tambah produk.

📘 Modul Praktikum

📖 Deskripsi Singkat

Mengenal Express.js lebih dalam: membuat server sederhana, memahami route dasar (GET, POST), dan memastikan server backend berjalan lancar.

🎯 Tujuan Pembelajaran

  • Mampu membuat server sederhana dengan Express.js.
  • Mampu mengimplementasikan struktur routing dasar (GET, POST).

👥 Pembagian Tugas Tim

  • Backend Engineer: Membuat & mengatur routing di Express.js.
  • Frontend Engineer: Mempelajari cara akses API (diimplementasikan kemudian).

🛠️ Langkah-Langkah Praktikum

a. Buat File index.js


// index.js
const express = require('express');
const cors = require('cors');
const app = express();
const PORT = 3001;

app.use(cors());
app.use(express.json());

app.get('/', (req, res) => {
  res.send('Hello World from Express.js!');
});

app.post('/data', (req, res) => {
  const { nama } = req.body;
  res.send(\`Data diterima: \${nama}\`);
});

app.listen(PORT, () => {
  console.log(\`Server berjalan di http://localhost:\${PORT}\`);
});

b. Jalankan Server


node index.js

Uji di browser: http://localhost:3001

c. Commit & Push


git add .
git commit -m "Add basic Express.js setup and routes"
git push origin main

d. Test GET & POST di Postman

  • GET: http://localhost:3001/
  • POST: http://localhost:3001/data (JSON body)

📊 Kriteria Penilaian

  • Server berjalan tanpa error (50%).
  • Route GET & POST berhasil diuji (50%).

✅ Checklist Praktikum

  • File index.js dibuat (GET, POST)
  • Server berjalan di http://localhost:3001
  • Commit & push ke GitHub

🧑‍💻 Latihan Mandiri

  1. Tambahkan route PUT atau DELETE.
  2. Uji endpoint baru dengan Postman.

🛠️ Troubleshooting Tips

  • Port Bentrok: Gunakan port lain atau hentikan proses existing.
  • Body Request Salah: Pastikan JSON & header Content-Type.
  • CORS Error: Pastikan app.use(cors()).
