Pertemuan 3: Koneksi Database PostgreSQL di Backend

🏢 Pentingnya Database

  • Menyimpan data secara terstruktur.
  • Mendukung proses CRUD (Create, Read, Update, Delete).
  • Berperan sebagai gudang informasi untuk server backend.

Analoginya seperti gudang penyimpanan

🔄 Alur Permintaan Data

  1. Client (Frontend) → Server (Backend)
  2. Server → Database (PostgreSQL)
  3. Database mengembalikan data ke Server → Client

⚙️ Peran PostgreSQL

  • Sistem manajemen database relasional andal.
  • Terintegrasi baik dengan Express.js lewat library pg.

PostgreSQL Logo

📘 Modul Praktikum

📖 Deskripsi Singkat

Menghubungkan Express.js dengan PostgreSQL memakai library pg, serta membuat query CRUD dasar (CREATE TABLE, SELECT, INSERT).

🎯 Tujuan Pembelajaran

  • Mengerti cara menghubungkan Express.js ke PostgreSQL.
  • Bisa melakukan operasi CRUD dasar (INSERT & SELECT).

👥 Pembagian Tugas Tim

  • Backend Engineer: Membuat koneksi DB & query dasar.
  • Frontend Engineer: Siapkan struktur data (untuk fetch di React).

🛠️ Langkah-Langkah Praktikum

a. Instal Library pg


npm install pg

b. Buat File db.js


// db.js
const { Pool } = require('pg');

const pool = new Pool({
  user: 'postgres',
  host: 'localhost',
  database: 'ecommerce-kecil',
  password: 'postgres',
  port: 5432,
});

module.exports = pool;

c. Inisialisasi Tabel


CREATE TABLE IF NOT EXISTS produk (
  id SERIAL PRIMARY KEY,
  nama VARCHAR(50),
  harga INT
);

d. Buat Route CRUD Dasar


const pool = require('./db');

// CREATE
app.post('/produk', async (req, res) => {
  // ...
});

// READ
app.get('/produk', async (req, res) => {
  // ...
});

e. Gabungkan Kode


// index.js (gabungan)
const express = require('express');
const cors = require('cors');
const pool = require('./db');

f. Test Endpoint

  • POST /produk
  • GET /produk

📊 Kriteria Penilaian

  • Koneksi PostgreSQL berhasil (30%).
  • CREATE & READ berjalan (70%).

✅ Checklist Praktikum

  • File db.js & koneksi DB sukses.
  • Tabel produk terbentuk.
  • Endpoint POST & GET berjalan baik.

🧑‍💻 Latihan Mandiri

  1. Tambahkan route DELETE dan UPDATE.
  2. Test dengan Postman, cek data di DB.

📅 Deadline & Pengumpulan

  • Akhir Pekan 3
  • Screenshot query + Link repo GitHub

🛠️ Troubleshooting Tips

  • ECONNREFUSED: Pastikan DB & host config benar.
  • Syntax Error: Cek penulisan SQL.
  • RETURNING: Pastikan PostgreSQL mendukung.
