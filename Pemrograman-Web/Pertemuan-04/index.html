Pertemuan 4: Dasar React.js (Vite) & Struktur Komponen

🔥 Mengapa React & Vite?

  • React: Library populer untuk UI dinamis.
  • Vite: Build tool yang cepat & ringan.

⚙️ Komponen & Hooks

  • useState & useEffect memudahkan state & lifecycle.
  • Komponen: memecah UI jadi bagian kecil reusable.

📘 Modul Praktikum

📖 Deskripsi Singkat

Fokus pada React.js (Vite): struktur folder, komponen dasar, dan state management sederhana (Hooks).

🎯 Tujuan Pembelajaran

  • Mengenal struktur proyek React + Vite.
  • Memahami konsep komponen & state di React.
  • Menggunakan Hooks (useState, useEffect).

👥 Pembagian Tugas Tim

  • Frontend Engineer: Membuat tampilan dasar dengan komponen + state.
  • Backend Engineer: Standby jika perlu penyesuaian endpoint.

🛠️ Langkah-Langkah Praktikum

a. Review Struktur Proyek Vite


nama-proyek-frontend
├─ public
├─ src
│  ├─ App.jsx
│  ├─ main.jsx
│  └─ ...
└─ index.html

b. Buat Komponen Dasar


// src/components/ProdukList.jsx
import React from 'react';

function ProdukList() {
  return (...);
}
export default ProdukList;

c. Impor Komponen di App.jsx


// src/App.jsx
import ProdukList from './components/ProdukList';

d. Gunakan useState & useEffect


import React, { useState, useEffect } from 'react';

e. Jalankan & Commit


npm run dev
git add .
git commit -m "Add ProdukList with state/effect"
git push origin main

📊 Kriteria Penilaian

  • React (Vite) berjalan tanpa error (30%).
  • Komponen tampil & data statis (40%).
  • Penggunaan useState & useEffect (30%).

✅ Checklist Praktikum

  • Folder components, minimal 1 komponen.
  • Data statis muncul di browser.
  • Commit & push berhasil.

🧑‍💻 Latihan Mandiri

  • Buat komponen lain (Header/Footer).
  • Coba styling sederhana (CSS).

🛠️ Troubleshooting Tips

  • Port Vite default: 5173, bukan 3000.
  • Blank Screen: Cek error di console browser.
  • Dependencies Error: Jalankan npm install.
