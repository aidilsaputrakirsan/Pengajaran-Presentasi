🎓 Pemrograman Web

Pertemuan 6: Form Input & Validasi Dasar

📝 Pentingnya Form

  • Menangkap input data pengguna (nama, harga, dsb.).
  • Validasi mencegah data kosong/invalid.

⚙️ Controlled Components

Menggunakan useState untuk mengontrol isi input.

📘 Modul Praktikum

📖 Deskripsi Singkat

Membuat form di React (Vite) untuk menambahkan data baru ke database via POST, plus validasi input dasar.

🎯 Tujuan Pembelajaran

  • Membuat form di React dengan controlled components.
  • Validasi sederhana sebelum kirim data ke backend.

👥 Pembagian Tugas Tim

  • Frontend Engineer: Komponen form, handle state & validasi.
  • Backend Engineer: Menyesuaikan endpoint POST jika perlu.

🛠️ Langkah-Langkah Praktikum

a. Buat Komponen TambahProduk.jsx


import React, { useState } from 'react';
import axios from 'axios';

b. Integrasi di App.jsx


// src/App.jsx
import TambahProduk from './components/TambahProduk';
import ProdukList from './components/ProdukList';

c. Uji Tambah Produk

  • Jalankan backend: node index.js
  • Jalankan frontend: npm run dev

📊 Kriteria Penilaian

  • Form dapat menambahkan data baru (50%).
  • Validasi input sederhana (30%).
  • Integrasi TambahProduk & ProdukList (20%).

✅ Checklist Praktikum

  • Komponen TambahProduk dibuat.
  • Validasi sederhana berfungsi.
  • Data baru muncul di database.

🧑‍💻 Latihan Mandiri

  • Notifikasi “Berhasil disimpan”/refresh otomatis daftar.
  • Styling form agar user-friendly.

📅 Deadline & Pengumpulan

  • Akhir Pekan 6
  • Screenshot form + data di DB + Link repo

🛠️ Troubleshooting Tips

  • Data Tidak Tersimpan: Cek body request di Network tab.
  • Error 500: Cek log server (field DB?).
  • Form Tidak Reset: Kosongkan state setelah POST.
