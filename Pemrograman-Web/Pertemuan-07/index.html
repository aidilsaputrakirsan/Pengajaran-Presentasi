Pertemuan 7: Penyempurnaan Fitur CRUD & Review

🔄 Pentingnya CRUD Lengkap

  • CRUD: Create, Read, Update, Delete → pondasi aplikasi data.
  • User dapat menambah, melihat, mengubah, dan menghapus data.

⚙️ Review Pekan 1–6

  • Pekan 1–3: Setup server, DB, Express.
  • Pekan 4–5: React (Vite), fetch data.
  • Pekan 6: Form input & validasi.

📘 Modul Praktikum

📖 Deskripsi Singkat

Menyelesaikan fitur CRUD (Create, Read, Update, Delete) secara penuh, review materi pekan 1–6 agar siap untuk aplikasi menengah.

🎯 Tujuan Pembelajaran

  • Menyempurnakan e-commerce kecil dengan CRUD lengkap.
  • Review integrasi React (Vite), Express, PostgreSQL.

👥 Pembagian Tugas Tim

  • Frontend Engineer: UI edit & delete produk.
  • Backend Engineer: Endpoint PUT & DELETE.

a. Endpoint PUT & DELETE


// index.js
app.put('/produk/:id', async (req, res) => { ... });
app.delete('/produk/:id', async (req, res) => { ... });

b. UI Update & Delete


// src/components/ProdukList.jsx
const handleDelete = (id) => { ... };

c. Review CRUD

  • POST, GET, PUT, DELETE berfungsi
  • Uji aplikasi e-commerce kecil

📊 Kriteria Penilaian

  • Fitur update & delete berfungsi (50%).
  • Full CRUD berjalan (50%).

✅ Checklist Praktikum

  • Endpoint PUT & DELETE di backend.
  • Edit & Delete di frontend.
  • Review tanpa error.

🧑‍💻 Latihan Mandiri

  1. Modal/form update data.
  2. Styling lebih menarik (CSS).

📅 Deadline & Pengumpulan

  • Akhir Pekan 7
  • Screenshot bukti CRUD + Link repo GitHub

🛠️ Troubleshooting Tips

  • Update Gagal: Periksa body request (nama/harga).
  • Delete Gagal: Pastikan ID valid.
  • Stale Data: Refresh GET setelah update/delete.
