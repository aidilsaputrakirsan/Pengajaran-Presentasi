🎯 Capaian Pembelajaran

  • Menguatkan pemahaman materi pekan 1–7.
  • Memperbaiki kekurangan aplikasi (bugfix) & menyusun dokumentasi dasar.

📖 Deskripsi Singkat

  • Memanfaatkan waktu untuk review konsep utama: instalasi, CRUD, routing, integrasi.
  • Fokus pada bug fixing, optimasi sederhana, & dokumentasi.

💡 Review Ringkas (Pekan 1–7)

  1. Pekan 1: Persiapan Lingkungan (Node.js, PostgreSQL, Git, React + Vite, Express)
  2. Pekan 2: Dasar Express.js & routing GET/POST
  3. Pekan 3: Koneksi PostgreSQL & query CRUD dasar
  4. Pekan 4: Dasar React.js (Vite) & struktur komponen + Hooks
  5. Pekan 5: Integrasi React-Express (fetch data GET)
  6. Pekan 6: Form input & validasi (POST ke backend)
  7. Pekan 7: CRUD lengkap (PUT, DELETE) & review singkat

🐞 Identifikasi Bug & Masalah Umum

  • Error saat fetch data karena endpoint tidak sesuai atau server mati.
  • Validasi kurang lengkap, user bisa input data kosong/ilegal.
  • Inconsistent state di React karena setState tidak diatur dengan baik.
  • Query database tidak aman (belum pakai parameterized query? Sudah? Periksa lagi!).

🔧 Strategi Bugfix

  1. Reproduksi bug & catat langkah-langkah jelas.
  2. Debugging tools: console.log, dev tools di browser, Postman, logs di server.
  3. Test case: buat skenario uji, pastikan tiap route & UI diuji.
  4. Perbaiki secara iteratif, lakukan *commit* terpisah untuk setiap perbaikan.

📄 Dokumentasi Dasar

  • **README**: Jelaskan cara *install*, *run*, & *deploy* (jika ada).
  • **Endpoint list**: GET `/produk`, POST `/produk`, dsb.
  • **Struktur folder**: Tunjukkan frontend/, backend/, db.js, dsb.
  • **ERD sederhana** (opsional): Gambaran schema tabel (users, produk, dsb.).

🧩 Latihan Mandiri

Beberapa ide latihan:

  • Buat fitur tambahan: misal kategori produk atau pencarian.
  • Implementasi validasi lebih ketat di sisi backend.
  • Eksperimen React Router agar navigasi UI lebih rapi.

Tujuan: memperdalam konsep & menyiapkan diri untuk proyek lebih besar.

📝 Contoh README yang Baik


# E-Commerce Mini (React + Express + PostgreSQL)

## Cara Instalasi
1. Clone repositori ini
2. Masuk folder `backend` dan jalankan `npm install`
3. Buat database PostgreSQL, lalu sesuaikan `db.js`
4. Jalankan `npm start` di folder `backend`
5. Masuk folder `frontend` dan jalankan `npm install`
6. Jalankan `npm run dev`

## Endpoint
- GET /produk => Menampilkan list produk
- POST /produk => Menambah produk baru
- PUT /produk/:id => Update produk
- DELETE /produk/:id => Hapus produk

## Struktur Folder
    .
    ├── frontend
    ├── backend
    │   ├── db.js
    │   ├── index.js
    │   └── ...
    └── README.md

...

*Pastikan* README selalu up-to-date dengan perubahan projek.
